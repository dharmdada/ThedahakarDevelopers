Enums & Records

Type-safe constants and immutable data carriers — two of Java's most useful features.

TH

Theory Deep Dive

Restricted values and transparent data carriers

Mental Model

Enums model a closed set of named values. They are safer than strings or integers because the compiler knows all valid options. An enum can also own fields and behavior, which keeps logic close to the value it belongs to.

Records model shallowly immutable data. They automatically provide constructor, accessors, equals, hashCode, and toString based on components. This makes them excellent for DTOs, value results, messages, and simple domain facts.

Enums and records both reduce accidental complexity. They make illegal states harder to represent and remove boilerplate that often hides the real intent of code.

Beginner-Friendly, Expert-Useful Lens

If you are new: Beginners should use enums instead of magic strings and records instead of boilerplate data carriers.

If you are revising: Experts should refresh enum behavior, record invariants, serialization concerns, pattern matching, and domain modeling limits.

Mastery check: You understand Enums and Records when you can explain the simple mental model, write a tiny runnable example, identify the common trap, and describe how it behaves in a real project.

Principles That Matter

  • Enums provide type safety for a fixed set of constants (e.g., DAYS_OF_WEEK).
  • Records (Java 14+) reduce boilerplate for immutable data carriers.
  • Enums can have fields, constructors, and methods, just like regular classes.

Common Misunderstandings

  • Using String constants (public static final String RED = "RED") instead of Enums, losing type safety.
  • Trying to instantiate an Enum using the 'new' keyword. (The JVM handles instantiation).
  • Assuming Records are just "less typing" for Java Beans. Records are strictly immutable; they don't have setters.

Mastery Checklist

  • You can create an Enum with a custom constructor and properties.
  • You understand the 'values()' and 'valueOf()' methods of Enums.
  • You know how to define a Record and access its components.

Hard-Won Tips

  • Enums can have fields, methods, and per-constant behavior; they are often cleaner than switch statements scattered across code.
  • Records are shallowly immutable. If a record contains a mutable List, the list can still change unless you copy it.
  • A compact record constructor is the right place to validate and normalize data before the object exists.
  • Do not use records for entities whose identity and lifecycle matter more than their current field values.
  • String statuses rot over time. Enums and sealed types make illegal domain states visible at compile time.

Enums: Type-Safe Constants

Imagine you are writing a program that processes orders, and an order can have a status: Pending, Shipped, or Delivered. In old languages, programmers would just use integer constants (1=Pending, 2=Shipped, 3=Delivered). The problem? Someone could pass 99 into your method, and your code would break. There was no Type Safety.

An enum (short for Enumeration) fixes this. It is a special Java class that represents a fixed, unchangeable set of values. If a method asks for an OrderStatus enum, it is physically impossible to pass it anything other than the exact values you defined.

JavaBasic Enum
enum Season { SPRING, SUMMER, AUTUMN, WINTER }

public class Main {
    public static void main(String[] args) {
        Season s = Season.SUMMER;

        // Switch with enum
        String msg = switch (s) {
            case SPRING -> "Flowers bloom";
            case SUMMER -> "Beach time";
            case AUTUMN -> "Leaves fall";
            case WINTER -> "Stay warm";
        };
        System.out.println(msg);

        // Useful methods
        Season[] all = Season.values();    // Array of all values
        Season fromStr = Season.valueOf("SUMMER"); // String → enum
        String name = s.name();             // "SUMMER"
        int ord = s.ordinal();              // 1 (position, 0-indexed)
        System.out.println(name + " at index " + ord);
    }
}

Advanced Enums: Not Just Labels

In languages like C, enums are just glorified integers. In Java, Enums are full-fledged Classes. This means an enum constant isn't just a label—it's an actual object. You can give these objects fields, constructors, and even custom behavior.

JavaPowerful Enum
enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    VENUS(4.869e+24, 6.0518e6),
    EARTH(5.976e+24, 6.37814e6),
    MARS(6.421e+23, 3.3972e6);

    private final double mass;    // in kg
    private final double radius;  // in meters

    // Enum constructor (always private)
    Planet(double mass, double radius) {
        this.mass = mass;
        this.radius = radius;
    }

    // Method
    public double surfaceGravity() {
        final double G = 6.67300E-11;
        return G * mass / (radius * radius);
    }

    public double surfaceWeight(double otherMass) {
        return otherMass * surfaceGravity();
    }
}

public class Main {
    public static void main(String[] args) {
        // Usage
        double weight = Planet.EARTH.surfaceWeight(75);
        System.out.println("Weight on Earth: " + weight);
    }
}

Enums Implementing Interfaces

Java
interface Printable {
    String toPrettyString();
}

enum Status implements Printable {
    ACTIVE {
        @Override
        public String toPrettyString() { return "✅ Active"; }
    },
    INACTIVE {
        @Override
        public String toPrettyString() { return "❌ Inactive"; }
    }
}

public class Main {
    public static void main(String[] args) {
        System.out.println(Status.ACTIVE.toPrettyString());
        System.out.println(Status.INACTIVE.toPrettyString());
    }
}

The Ultimate Singleton Pattern

A "Singleton" is an architectural pattern that guarantees a class only ever has ONE instance in the entire application (e.g., a Database Connection Pool). Writing a safe Singleton in Java is notoriously difficult because of multithreading and serialization edge cases.

Joshua Bloch (author of Effective Java) stated that a single-element Enum is the absolute best way to implement a Singleton in Java. The JVM natively guarantees that an Enum value is instantiated exactly once, is thread-safe, and survives serialization perfectly.

JavaEnum Singleton
enum DatabaseConnection {
    INSTANCE; // The one and only instance

    public void connect() {
        System.out.println("Connected to DB securely.");
    }
}

public class Main {
    public static void main(String[] args) {
        // Usage:
        DatabaseConnection.INSTANCE.connect();
    }
}

Records: The End of Boilerplate

For decades, Java developers complained about how much code it took to create a simple "Data Carrier" class. To make a class that just held an x and y coordinate, you had to write the fields, write the constructor, write the getters, override equals(), override hashCode(), and override toString(). A tiny concept required 40 lines of code.

Java 14 introduced Records. A Record is a special type of class designed strictly to hold immutable data. You provide the components (fields) in the declaration, and the Java compiler magically generates the constructor, the accessors, and the foundational Object methods for you behind the scenes.

JavaRecord vs Class
// ❌ Old way: ~30 lines of boilerplate
class PointClass {
    private final int x, y;
    public PointClass(int x, int y) { this.x = x; this.y = y; }
    public int x() { return x; }
    public int y() { return y; }
    // + equals, hashCode, toString...
}

// ✅ Record: ONE LINE does it all
record Point(int x, int y) {}

public class Main {
    public static void main(String[] args) {
        // Auto-generated:
        Point p = new Point(3, 5);
        System.out.println(p.x());          // 3 (accessor, no 'get' prefix!)
        System.out.println(p.y());          // 5
        System.out.println(p.toString());   // "Point[x=3, y=5]"
        System.out.println(p.equals(new Point(3, 5))); // true
    }
}

Record Features

JavaAdvanced Records
record User(String name, String email, int age) {
    public User {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("name is required");
        }
        if (email == null || !email.contains("@")) {
            throw new IllegalArgumentException("valid email is required");
        }
        if (age < 0) {
            throw new IllegalArgumentException("age cannot be negative");
        }
    }

    public boolean isAdult() {
        return age >= 18;
    }
}

public class Main {
    public static void main(String[] args) {
        User user = new User("Alice", "alice@example.com", 25);
        System.out.println(user);
        System.out.println("Adult? " + user.isAdult());
    }
}

Destructuring: Record Patterns (Java 21)

Because Records transparently carry data, modern Java (21+) allows you to "destructure" them directly inside a switch statement or instanceof check. This extracts the fields into local variables automatically, making data processing incredibly clean.

JavaRecord Pattern Matching
record Point(int x, int y) {}
record Circle(Point center, int radius) {}

public class Main {
    public static void main(String[] args) {
        Object shape = new Circle(new Point(10, 20), 5);

        if (shape instanceof Circle circle) {
            Point center = circle.center();
            int radius = circle.radius();
            System.out.println("Circle at " + center.x() + "," + center.y());
            System.out.println("Radius: " + radius);
        }
    }
}

Real-World Use Case: State Machines and DTOs

Enums are perfect for state machines. If you are building a food delivery app like UberEats, the order status (PLACED, COOKING, DELIVERING, COMPLETED) is an Enum. This prevents a developer from accidentally typing "cooking" with a lowercase 'c', which would break the system. It guarantees type safety.

Records are the ultimate choice for DTOs (Data Transfer Objects). When you fetch a user's profile from a database or receive it via a REST API, you don't need to modify that data; you just need to carry it. A Record allows you to define that carrier in a single line, keeping your codebase clean and readable.

Interview Preparation

Question 1
Are Enums thread-safe?
Yes, the JVM guarantees that Enums are instantiated only once. This makes them a perfect and simple way to implement the Singleton design pattern.
Question 2
Can a Record implement an Interface?
Yes. Records cannot extend other classes (they implicitly extend java.lang.Record), but they can implement interfaces.
