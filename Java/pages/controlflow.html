Control Flow

Decision making, loops, and flow control — every variation.

TH

Theory Deep Dive

Control flow as decision architecture

Mental Model

Control flow decides which code runs and how often. Every if statement expresses a business rule, every loop expresses repetition, and every switch represents a finite set of alternatives. Clear control flow makes program behavior predictable.

Branches should encode meaningful cases, not accidental implementation details. Deeply nested conditions often signal that a method has too many responsibilities or that guard clauses would expose the main path more clearly.

Loops require a termination story. A loop has an initial state, a condition, progress toward completion, and a body. When one of these is unclear, bugs appear as infinite loops, skipped elements, duplicated work, or off-by-one errors.

Principles That Matter

  • Keep loops and conditionals simple. Extract complex logic into well-named methods.
  • Use the enhanced for-loop whenever you don't need the index.
  • Switch statements (especially Java 14+ switch expressions) are cleaner than long if-else chains.

Common Misunderstandings

  • Forgetting the 'break' statement in a traditional switch, causing unintended fall-through.
  • Using a while loop when a for loop is more appropriate (e.g., when you know the number of iterations).
  • Modifying a collection while iterating over it with a standard for-each loop (causes ConcurrentModificationException).

Mastery Checklist

  • You can refactor a long if-else chain into a modern switch expression.
  • You know when to use a do-while loop vs a while loop.
  • You understand how to safely break out of nested loops using labels.

If / Else If / Else: Branching Logic

By default, Java executes code line by line, from top to bottom. Control flow statements allow you to branch off that path based on conditions. The most fundamental branching mechanism is the if statement.

Think of if statements as a train track switch. If the condition inside the parentheses evaluates to true, the train goes down the first track (the first block of code). If it is false, the train continues to the next else if or else block.

Java
int score = 85;

if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");
} else if (score >= 70) {
    System.out.println("Grade: C");
} else {
    System.out.println("Grade: F");
}

// Nested if
if (age >= 18) {
    if (hasLicense) {
        System.out.println("Can drive");
    }
}

Switch Statement (Classic)

When you have a single variable and you want to check it against many possible exact values, a long chain of if / else if becomes very hard to read. The switch statement provides a cleaner alternative.

In older versions of Java, the switch statement had a "fall-through" behavior inherited from C. This meant if you didn't explicitly write break; at the end of a case, it would execute the next case below it automatically, which caused countless bugs.

Java
String day = "MONDAY";
switch (day) {
    case "MONDAY": case "TUESDAY": case "WEDNESDAY":
    case "THURSDAY": case "FRIDAY":
        System.out.println("Weekday");
        break; // Without break, it falls through!
    case "SATURDAY": case "SUNDAY":
        System.out.println("Weekend");
        break;
    default:
        System.out.println("Invalid");
}

Switch Expressions: The Modern Upgrade

To fix the notorious "fall-through" bugs of the classic switch, Java 14 introduced Switch Expressions. This modern syntax uses an arrow (->) instead of a colon.

  • No Fall-Through: You don't need break; anymore. Only the matched block executes.
  • Returns a Value: The entire switch block can be assigned to a variable.
  • Exhaustiveness: If you are returning a value, the compiler forces you to cover every possible case (or provide a default).
Java
// Arrow syntax — no break needed, no fall-through, returns a value
String type = switch (day) {
    case "MONDAY", "TUESDAY", "WEDNESDAY",
         "THURSDAY", "FRIDAY"    -> "Weekday";
    case "SATURDAY", "SUNDAY" -> "Weekend";
    default                    -> "Invalid";
};

// Multi-line case with yield
int len = switch (day) {
    case "MONDAY" -> 6;
    default -> {
        String trimmed = day.trim();
        yield trimmed.length();
    }
};

// Pattern matching in switch
Object obj = 42;
String result = switch (obj) {
    case Integer i -> "Integer: " + i;
    case String s  -> "String: " + s;
    case null     -> "null!";
    default       -> "Other";
};

The Traditional For Loop

When you know exactly how many times you want a block of code to run, you use a for loop. It is composed of three parts inside the parentheses, separated by semicolons:

  1. Initialization: Executes exactly once before the loop starts (e.g., int i = 0).
  2. Condition: Checked before every iteration. If true, the loop runs (e.g., i < 10).
  3. Increment: Executes exactly once after every iteration (e.g., i++).
Java
// Standard
for (int i = 0; i < 10; i++) { System.out.println(i); }

// Counting down
for (int i = 10; i >= 0; i--) { }

// Multiple variables
for (int i = 0, j = 10; i < j; i++, j--) { }

// Infinite loop
for (;;) { break; }

Enhanced For-Each

Java
String[] fruits = {"Apple", "Banana", "Cherry"};
for (String fruit : fruits) {
    System.out.println(fruit);
}

// Works with any Iterable (List, Set, etc.)
List<Integer> nums = List.of(10, 20, 30);
for (int n : nums) { System.out.println(n); }

// Map iteration
Map<String, Integer> ages = Map.of("Alice", 25);
for (var entry : ages.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

While & Do-While: Unknown Iterations

What if you don't know how many times a loop should run? For example, "keep reading the file until there is no more text." In this case, you use a while loop.

The difference between while and do-while is simply when the condition is checked. A while loop checks before running, so it might run zero times. A do-while loop runs the code first, and then checks the condition, guaranteeing it runs at least once.

Java
// While: checks BEFORE each iteration
int count = 5;
while (count > 0) {
    System.out.println(count--);
}

// Do-While: runs AT LEAST ONCE
int x = 0;
do {
    System.out.println("Runs once even though x is 0");
} while (x > 0);

Break, Continue & Labels

Java
// break: exits the ENTIRE loop
for (int i = 0; i < 100; i++) {
    if (i == 5) break; // Stops at 5
}

// continue: skips CURRENT iteration
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) continue; // Skip evens
    System.out.println(i); // 1,3,5,7,9
}

// Labels: break/continue an OUTER loop from inside a nested loop
outer:
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 5; j++) {
        if (i * j > 6) break outer; // Breaks BOTH loops
    }
}

Real-World Use Case: Game Loops

Every video game runs on a massive while loop, often called the "Game Loop".

The loop runs indefinitely: while (gameIsRunning). Inside the loop, it reads controller input, updates the physics engine, checks for collisions, and renders the graphics. Only when the player hits "Quit Game" does gameIsRunning become false, breaking the loop and shutting down the application. Without control flow structures, a game would execute once and immediately exit!

Interview Preparation

Question 1
Can you use a String in a switch statement?
Yes, since Java 7, Strings are supported in switch statements. Under the hood, it uses the String's hash code.
Question 2
What is the 'continue' keyword?
It skips the rest of the current loop iteration and proceeds immediately to the next iteration.
