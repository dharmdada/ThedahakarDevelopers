Theory Deep Dive

Streams as pipes — connecting a source to a sink

Mental Model: The Two Hierarchies

Java I/O has two completely separate hierarchies for two different units of data:

Byte Streams
InputStream / OutputStream
Raw binary data — 8 bits at a time. Use for: images, audio, video, compressed files, network data, serialized objects. Everything is bytes at this level.
Character Streams
Reader / Writer
Text data — characters using a specific charset (UTF-8, etc). Use for: text files, CSV, JSON, XML, log files, config files. Handles encoding/decoding for you.
Bridge Classes
InputStreamReader / OutputStreamWriter
Convert between byte and character worlds. You specify the charset. Critical when reading bytes from network/file and treating them as text.
Modern Alternative
java.nio.file.Files
For most file operations, NIO.2 is simpler. But the classic stream API is still essential for network sockets, HTTP bodies, serialization, and compression.

All java.io classes implement the Decorator pattern. You stack wrappers around a base stream to add capabilities: buffering, charset conversion, data type reading, compression. Each wrapper calls the one beneath it.

JavaThe Decorator chain — reading a gzipped text file
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.util.zip.GZIPOutputStream;
import java.util.zip.GZIPInputStream;

public class Main {
    public static void main(String[] args) throws IOException {
        try (var out = new GZIPOutputStream(new FileOutputStream("data.log.gz"))) {
            out.write("first line\nsecond line\n".getBytes(StandardCharsets.UTF_8));
        }

        try (var br = new BufferedReader(
                new InputStreamReader(
                    new GZIPInputStream(new FileInputStream("data.log.gz")),
                    StandardCharsets.UTF_8))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        }
    }
}
ℹ️
Always wrap with BufferedReader/BufferedWriter. Unbuffered reads hit the OS for every single character. Buffering reads 8KB chunks at a time — often 100x faster for text processing.

The Full Stream Hierarchy

ℹ️
Byte Streams
InputStream (abstract)
├── FileInputStream — bytes from a file
├── ByteArrayInputStream — bytes from a byte[] in memory
├── FilterInputStream (decorator base)
│    ├── BufferedInputStream — adds buffering
│    ├── DataInputStream — reads primitives (int, long, double)
│    └── CheckedInputStream — computes checksum while reading
├── ObjectInputStream — deserializes Java objects
└── GZIPInputStream / ZipInputStream — decompression

OutputStream (abstract)
├── FileOutputStream — bytes to a file
├── ByteArrayOutputStream — bytes to a byte[] in memory
├── FilterOutputStream
│    ├── BufferedOutputStream — buffers writes
│    ├── DataOutputStream — writes primitives
│    └── PrintStream — System.out is this!
└── ObjectOutputStream — serializes Java objects

Character Streams
Reader (abstract)
├── InputStreamReader — bridge: bytes → chars (specify charset!)
│    └── FileReader — shortcut for FileInputStream + InputStreamReader
├── BufferedReader — buffers + readLine()
├── StringReader — reads from a String
└── CharArrayReader — reads from a char[]

Writer (abstract)
├── OutputStreamWriter — bridge: chars → bytes (specify charset!)
│    └── FileWriter — shortcut for FileOutputStream + OutputStreamWriter
├── BufferedWriter — buffers + newLine()
├── PrintWriter — println, printf, format
└── StringWriter — writes to a String (via StringBuffer internally)

Byte Streams — FileInputStream / FileOutputStream

JavaBinary file copy — the complete, correct approach
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.nio.file.*;

public class Main {
    public static void main(String[] args) throws IOException {
        Path src = Path.of("source.bin");
        Path dest = Path.of("dest.bin");
        Files.write(src, "binary-like content".getBytes(StandardCharsets.UTF_8));

        try (var in = new BufferedInputStream(new FileInputStream(src.toFile()));
             var out = new BufferedOutputStream(new FileOutputStream(dest.toFile()))) {
            byte[] buffer = new byte[8192];
            int bytesRead;
            while ((bytesRead = in.read(buffer)) != -1) {
                out.write(buffer, 0, bytesRead);
            }
        }

        try (var in = new FileInputStream(src.toFile());
             var out = new ByteArrayOutputStream()) {
            long transferred = in.transferTo(out);
            System.out.println("Transferred bytes: " + transferred);
            System.out.println(new String(out.toByteArray(), StandardCharsets.UTF_8));
        }
    }
}

Character Streams — BufferedReader / BufferedWriter

JavaText file reading — all correct patterns
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.nio.file.*;

public class Main {
    static void processLine(String line) {
        System.out.println("Line: " + line);
    }

    public static void main(String[] args) throws IOException {
        Path file = Path.of("report.csv");
        Files.writeString(file, "name,score\nAlice,90\nBob,82\n", StandardCharsets.UTF_8);

        try (var br = new BufferedReader(
                new InputStreamReader(new FileInputStream(file.toFile()), StandardCharsets.UTF_8))) {
            String line;
            while ((line = br.readLine()) != null) {
                processLine(line);
            }
        }

        try (var br = Files.newBufferedReader(file, StandardCharsets.UTF_8)) {
            long aliceLines = br.lines().filter(line -> line.contains("Alice")).count();
            System.out.println("Alice lines: " + aliceLines);
        }
    }
}
JavaWriting text — BufferedWriter / PrintWriter
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.nio.file.*;

public class Main {
    public static void main(String[] args) throws IOException {
        try (var bw = new BufferedWriter(
                new OutputStreamWriter(new FileOutputStream("log.txt", true), StandardCharsets.UTF_8))) {
            bw.write("2026-05-20 INFO Server started");
            bw.newLine();
            bw.write("2026-05-20 INFO Listening on port 8080");
            bw.newLine();
        }

        try (var pw = new PrintWriter(new BufferedWriter(
                new OutputStreamWriter(new FileOutputStream("output.txt"), StandardCharsets.UTF_8)))) {
            pw.println("Hello World");
            pw.printf("User: %s, Age: %d%n", "Alice", 30);
            pw.format("Score: %.2f%n", 98.567);
            if (pw.checkError()) {
                throw new IOException("PrintWriter failed");
            }
        }

        System.out.println(Files.readString(Path.of("output.txt"), StandardCharsets.UTF_8));
    }
}

Scanner — Parsing Tokens from Text

Scanner is not a stream — it's a parser that reads tokens. It works on top of any InputStream, Reader, File, or String.

JavaScanner — parsing structured text input
import java.nio.charset.StandardCharsets;
import java.nio.file.*;
import java.util.*;
import java.util.regex.MatchResult;

public class Main {
    public static void main(String[] args) throws Exception {
        String input = "Alice\n30\n";
        try (var sc = new Scanner(input)) {
            String name = sc.nextLine();
            int age = sc.nextInt();
            System.out.printf("Hello %s, you are %d years old.%n", name, age);
        }

        Path numbers = Path.of("numbers.txt");
        Files.writeString(numbers, "10 20 30", StandardCharsets.UTF_8);
        try (var sc = new Scanner(numbers, StandardCharsets.UTF_8)) {
            int sum = 0;
            while (sc.hasNextInt()) sum += sc.nextInt();
            System.out.println("Sum: " + sum);
        }

        try (var sc = new Scanner("Alice,30,Engineer").useDelimiter(",")) {
            System.out.println(sc.next() + " is " + sc.nextInt() + " and works as " + sc.next());
        }

        try (var sc = new Scanner("Price: $42.99 and $15.00")) {
            sc.findAll("\\$(\\d+\\.\\d{2})")
              .map(MatchResult::group)
              .forEach(System.out::println);
        }
    }
}
WARN
nextInt() + nextLine() trap. nextInt() reads the number but leaves the \n in the buffer. The very next nextLine() reads that empty leftover line, not the user's next input. Fix: call sc.nextLine() once after any nextInt() / nextDouble() to consume the newline.

DataInputStream / DataOutputStream — Typed Binary Protocol

JavaWriting and reading a binary protocol
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
        try (var dos = new DataOutputStream(
                new BufferedOutputStream(new FileOutputStream("data.bin")))) {
            dos.writeInt(42);
            dos.writeLong(123456789L);
            dos.writeDouble(3.14159);
            dos.writeBoolean(true);
            dos.writeUTF("Hello");
        }

        try (var dis = new DataInputStream(
                new BufferedInputStream(new FileInputStream("data.bin")))) {
            int num = dis.readInt();
            long id = dis.readLong();
            double pi = dis.readDouble();
            boolean flag = dis.readBoolean();
            String message = dis.readUTF();
            System.out.printf("%d %d %.5f %s %s%n", num, id, pi, flag, message);
        }
    }
}

Object Serialization — ObjectInputStream / ObjectOutputStream

JavaSerialize and deserialize a Java object
import java.io.*;

class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private final String name;
    private final int age;
    private transient String password;

    User(String name, int age) {
        this.name = name;
        this.age = age;
        this.password = "default123";
    }

    String name() { return name; }
    String password() { return password; }

    @Override public String toString() { return name + " (" + age + ")"; }
}

public class Main {
    public static void main(String[] args) throws Exception {
        User user = new User("Alice", 30);
        try (var oos = new ObjectOutputStream(
                new BufferedOutputStream(new FileOutputStream("user.ser")))) {
            oos.writeObject(user);
        }

        try (var ois = new ObjectInputStream(
                new BufferedInputStream(new FileInputStream("user.ser")))) {
            User loaded = (User) ois.readObject();
            System.out.println(loaded);
            System.out.println("Password after deserialize: " + loaded.password());
        }
    }
}
WARN
Java serialization is a security risk. Deserializing untrusted data can lead to remote code execution (the "billion laughs" attack and gadget chains). In modern Java, prefer JSON (Jackson), Protocol Buffers, or Java Records for data transfer. Use serialization only for internal, trusted data.

Principles That Matter

  • Always specify charset explicitly — FileReader uses the platform default charset. On Windows that's often CP1252, not UTF-8. Always use InputStreamReader(stream, StandardCharsets.UTF_8).
  • Always buffer your streams — Raw FileInputStream/FileOutputStream are unbuffered. Wrap with Buffered* for any non-trivial I/O.
  • Always close in try-with-resources — Streams hold OS file descriptors. Unclosed streams = file descriptor leak.
  • Close outermost stream only — Closing the outermost wrapper closes all inner streams. Don't close them individually.
  • Flush before close — Buffered streams may hold data. Closing calls flush() but if you need data written immediately (e.g. network), call flush() explicitly.

Common Misunderstandings

  • ⚠ FileReader uses platform default charset — This makes your code platform-dependent. Always use InputStreamReader + explicit charset instead.
  • ⚠ FileWriter(file, true) appends, false overwrites — The boolean is "append mode". Default (no boolean) is overwrite. Many bugs come from not knowing this.
  • ⚠ read() returns int, not char — Returns -1 at end-of-stream. Casting to char before checking for -1 is a bug: (char)-1 == '\uFFFF'.
  • ⚠ PrintWriter never throws IOException — It suppresses them! Call checkError() to detect write failures, or use a Writer that throws.

Hard-Won Tips

  • Never read a large file into memory by habit. Choose streaming, chunking, or memory mapping based on size and access pattern.
  • Buffered streams are not optional decoration; they can turn many tiny system calls into efficient bulk transfer.
  • Text files require an explicit charset. UTF-8 should be deliberate, not accidental.
  • Closing the outer wrapper usually closes the underlying stream; understand ownership when composing streams.
  • NIO channels and Files APIs often produce cleaner, safer code than old manual stream loops.

Interview Preparation

Question 1
Byte streams vs Character streams?
Byte streams (InputStream/OutputStream) work with raw 8-bit data — use for binary files, images, audio. Character streams (Reader/Writer) work with chars using a specific encoding — use for text. The bridge classes (InputStreamReader, OutputStreamWriter) convert between them with an explicit charset.
Question 2
Why wrap streams in Buffered*?
Unbuffered reads/writes call the OS for every byte — a system call per character. BufferedReader reads 8KB at a time into memory, then serves your reads from that buffer. Result: 100x fewer system calls. Always wrap file/network streams in Buffered* for any text processing.
Question 3
What design pattern does java.io use?
The Decorator pattern. Each stream wraps another and adds one capability (buffering, charset conversion, compression, type parsing). You compose features by stacking wrappers: BufferedReader(InputStreamReader(GZIPInputStream(FileInputStream))) adds buffering + charset + decompression to a file source.
Question 4
Why is FileReader dangerous?
FileReader uses the JVM's platform default charset (Charset.defaultCharset()). On Windows that's often CP1252; on Linux, UTF-8. Code that works on your machine breaks on the server. Always use new InputStreamReader(new FileInputStream(f), StandardCharsets.UTF_8) or in Java 11+: new FileReader(f, StandardCharsets.UTF_8).
