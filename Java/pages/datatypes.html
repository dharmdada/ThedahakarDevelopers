Data Types & Variables

Everything about how Java stores and represents data.

ℹ️
Statically Typed LanguageEvery variable in Java must be declared with a type before use. The type determines what values the variable can hold and what operations can be performed on it.
TH

Theory Deep Dive

Types as contracts between code and compiler

Mental Model

A type tells Java what values are allowed and what operations are legal. This is more than storage size. It is a contract that lets the compiler reject impossible operations before the program runs.

Primitive types represent raw values such as numbers, booleans, and characters. Reference types point to objects on the heap. Understanding this difference explains why primitives compare by value, object variables can be null, arrays know their length, and wrappers are needed when a collection requires objects.

Numeric types deserve special care because overflow, precision, casting, and promotion rules can silently change results. int is usually the default for whole numbers, long for larger counts, BigDecimal for money, and double for scientific or approximate measurement.

Principles That Matter

  • Primitives live on the stack (fast), Objects live on the heap (overhead).
  • Floating point arithmetic is not precise. Use BigDecimal for financial calculations.
  • Understand implicit casting (widening) vs explicit casting (narrowing).

Common Misunderstandings

  • Assuming == compares values for wrapper classes. It compares references. Use .equals().
  • Using float or double for money. They suffer from rounding errors.
  • Forgetting that char is essentially a 16-bit integer (UTF-16 code unit) in Java.

Mastery Checklist

  • You know the exact memory size of int, long, double, and boolean.
  • You can predict overflow behavior when adding two large ints.
  • You know when to use Integer vs int.

Primitive Data Types: The Building Blocks

Imagine your computer's memory as a massive wall of tiny, physical mailboxes. A primitive data type is like taking a piece of paper, writing a simple number or letter on it, and shoving it directly into one of those mailboxes.

Because they are stored directly in memory (on the Stack) and don't require any complex object overhead, primitives are incredibly fast and memory-efficient. Java is not a "pure" object-oriented language specifically because it retains these 8 primitive types for performance.

Reference Types: Remote Controls

If a primitive is a piece of paper in a mailbox, a Reference Type is like a remote control for a television. The variable itself does not contain the data; it only contains the memory address (the remote control signal) pointing to where the massive, complex Object actually lives in the Heap.

Whenever you use the new keyword, you are creating a new Object in the Heap. The variable you assign it to simply holds the reference. This is why multiple variables can point to the exact same object—they are just multiple remote controls pointing to the same TV.

JavaPrimitives vs References
// Primitive: holds actual value
int a = 10;
int b = a;  // b gets a COPY of the value
b = 20;     // a is still 10

// Reference: holds a pointer to an object
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;  // arr2 points to the SAME array
arr2[0] = 99;       // arr1[0] is NOW 99 too!

// Common reference types
String name = "Alice";           // String object
int[] numbers = new int[5];      // Array object
Scanner sc = new Scanner(System.in); // Scanner object

// null means "points to nothing"
String nothing = null;
// nothing.length()  →  NullPointerException!

Financial Types: BigInteger & BigDecimal

Never, ever use float or double for currency. Floating-point numbers are stored as binary fractions, meaning they cannot precisely represent decimal fractions like 0.1. Over thousands of calculations, pennies will mysteriously vanish or appear.

For high-precision, arbitrary-length numbers, Java provides the immutable BigDecimal and BigInteger classes.

JavaBigDecimal Example
// ❌ BAD: Floating point rounding error
double total = 0.1 + 0.2;
System.out.println(total); // Prints: 0.30000000000000004

// ✅ GOOD: BigDecimal precision
BigDecimal b1 = new BigDecimal("0.1"); // ALWAYS use String constructor
BigDecimal b2 = new BigDecimal("0.2");
BigDecimal result = b1.add(b2);
System.out.println(result); // Prints: 0.3

Type Casting: Forcing Conversions

Sometimes you need to fit a square peg into a round hole. Type casting is how you tell the Java compiler to convert a value of one data type into another. Think of it like pouring water between different sized buckets.

There are two types of casting in Java:

Widening (Implicit)
Automatic
Small → Large type. No data loss. Done by compiler.
Narrowing (Explicit)
Manual
Large → Small type. Possible data loss. You must cast.
JavaType Casting
// Widening: byte → short → int → long → float → double
int myInt = 9;
double myDouble = myInt;  // Automatic: 9.0

// Narrowing: double → float → long → int → short → byte
double pi = 3.14159;
int truncated = (int) pi;  // Manual cast: 3 (decimal part lost!)

// Be careful with overflow!
int big = 130;
byte small = (byte) big; // -126! (wraps around because byte max is 127)

// String to Number conversions
String numStr = "42";
int num = Integer.parseInt(numStr);
double dbl = Double.parseDouble("3.14");

// Number to String
String str = String.valueOf(42);
String str2 = 42 + ""; // Quick trick

Constants & final

Use the final keyword to make a variable's value unchangeable after assignment. Convention: use ALL_UPPERCASE for constant names.

JavaConstants
final int MAX_USERS = 100;
final double TAX_RATE = 0.18;
final String APP_NAME = "JavaMaster";

// MAX_USERS = 200;  →  Compile Error! Cannot reassign final variable

// final with reference types: the reference can't change, but the object CAN
final List<String> list = new ArrayList<>();
list.add("Hello");  // ✅ OK: modifying the object
// list = new ArrayList<>();  →  ❌ Error: reassigning the reference

var – Local Variable Type Inference

The var keyword lets the compiler infer the type of a local variable from its initializer. The variable is still strongly typed — var is NOT dynamic typing.

JavaUsing var
var name = "Alice";      // Compiler infers String
var count = 42;           // Compiler infers int
var prices = new ArrayList<Double>(); // Infers ArrayList<Double>
var map = Map.of("a", 1); // Infers Map<String, Integer>

// Where you CANNOT use var:
// ❌ var x;              (no initializer)
// ❌ var x = null;       (can't infer type from null)
// ❌ class fields        (only local variables)
// ❌ method parameters   (only local variables)
💡
When to Use varUse var when the type is obvious from the right side (e.g., var list = new ArrayList<String>()). Avoid it when the type isn't clear (e.g., var result = process(); — what type is result?).

Wrapper Classes: Making Primitives Object-Oriented

Java's Collections Framework (like ArrayList or HashMap) and Generics completely ignore primitives. You cannot create an ArrayList<int> because an ArrayList is designed to hold Objects (remote controls), not raw primitive values.

To bridge this gap, Java provides a Wrapper Class for every single primitive. These classes literally "wrap" a primitive value inside a full-fledged Object. For example, the Integer class is just a normal Java object that holds a single int inside it.

PrimitiveWrapperExample

JavaAutoboxing & Unboxing
// Autoboxing: primitive → Wrapper (automatic)
Integer boxed = 42;   // int → Integer automatically

// Unboxing: Wrapper → primitive (automatic)
int unboxed = boxed;   // Integer → int automatically

// Why it matters: Collections need objects
List<Integer> nums = new ArrayList<>();
nums.add(10);  // Autoboxing: int 10 → Integer.valueOf(10)
int first = nums.get(0); // Unboxing: Integer → int

// Useful Wrapper methods
int max = Integer.MAX_VALUE;         // 2,147,483,647
int parsed = Integer.parseInt("123"); // String → int
String binary = Integer.toBinaryString(255); // "11111111"
🔥
NullPointerException Trap!Wrapper types can be null, primitives cannot. Unboxing a null wrapper throws a NullPointerException: Integer x = null; int y = x; → 💥 NPE!

Variable Scope

Scope defines where a variable can be accessed. Java has four levels of scope:

JavaVariable Scopes
public class ScopeDemo {

    // 1. CLASS/INSTANCE SCOPE — accessible anywhere in the class
    private int instanceVar = 10;
    private static int classVar = 20; // Shared across all instances

    public void myMethod() {
        // 2. METHOD/LOCAL SCOPE — only inside this method
        int localVar = 30;

        if (true) {
            // 3. BLOCK SCOPE — only inside this if-block
            int blockVar = 40;
        }
        // blockVar is NOT accessible here!

        for (int i = 0; i < 5; i++) {
            // 4. LOOP SCOPE — 'i' only exists inside this loop
        }
        // i is NOT accessible here!
    }
}

Real-World Use Case: Database Mapping

When connecting a Java application to a relational database (like MySQL or PostgreSQL) using an ORM like Hibernate, data types are mapped directly.

A database table might have a column is_active which allows NULL values (because the status is unknown). In Java, if you map this to a primitive boolean, the application will crash when it tries to load a NULL because primitives cannot be null! To fix this, developers use the Wrapper class Boolean, which can safely hold true, false, or null.

Interview Preparation

Question 1
Primitive vs Wrapper Class?
Primitives (int, double) are raw values stored on the Stack (fast, memory efficient). Wrapper classes (Integer, Double) are full objects stored on the Heap and can be used in Collections.
Question 2
What is Autoboxing?
It is the automatic conversion that the Java compiler makes between the primitive types and their corresponding object wrapper classes (e.g., converting int to Integer).
Question 3
Why can't 'var' be used for class fields?
Because var relies on type inference from the right side of the equals sign at the moment of declaration. Class fields have default values and their types need to be known by the compiler before any code runs.
