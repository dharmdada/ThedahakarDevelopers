Install Java correctly, configure tools, and avoid common setup mistakes before coding.

Theory Deep Dive

Environment setup as a reliability problem

Mental Model

Setting up Java is not just installing a program. It establishes the toolchain that will compile, test, package, and run everything you build. When the terminal, IDE, and build tool point to different JDKs, errors become confusing because each layer sees a different world.

A correct setup has three parts: the JDK installation, the environment variables that expose the JDK to tools, and project configuration that pins the expected language level. Build tools can also use Java toolchains so the project remains reproducible across machines.

Beginners often think setup is a one-time chore. In real projects, setup is part of engineering hygiene. The README, wrapper scripts, CI configuration, and IDE metadata all work together to make development repeatable.

Install a JDK (Not Just Runtime)

Install an LTS version (for example Java 17 or 21). Ensure terminal commands resolve to the same version your IDE uses.

Shell
java -version
javac -version

Environment Variables

  • Set JAVA_HOME to your JDK root.
  • Add %JAVA_HOME%/bin (or equivalent) to PATH.
  • Restart terminal after changes and verify again.
WARN
Most setup bugs come from path conflicts. If multiple Java versions are installed, check command precedence and IDE runtime settings.

Your First Program

JavaHelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
Shell
javac HelloWorld.java
java HelloWorld

IDE Setup Checklist

  • Set project SDK explicitly (avoid "default" ambiguity).
  • Enable auto-import and formatting rules.
  • Use build tool wrapper where possible (gradlew / mvnw).
  • Configure JUnit test runner before writing many tests.

Common Errors and Fixes

Error
javac not found
PATH is missing JDK bin folder.
Error
Class not found
Class name or compile command mismatch.
Error
Version mismatch
IDE and terminal use different JDK versions.
