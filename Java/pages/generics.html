Generics Deep Dive (<T>)

Type Erasure, Wildcards, and the PECS Principle.

TH

Theory Deep Dive

Type parameters as compile-time safety

Mental Model

Generics let classes and methods work with a type chosen by the caller while preserving compile-time safety. A List<String> and List<Integer> use the same implementation idea but have different compile-time contracts.

Type erasure means generic information is mostly removed at runtime. This keeps old Java compatibility but explains restrictions such as not creating new T[] directly and not checking instanceof List<String>.

Wildcards describe flexibility at API boundaries. extends means the collection produces values you can read as a base type. super means the collection can consume values of a specific subtype.

The Problem: Type Safety

Before Java 5, collections held pure Object references. If you created a List, you could put a String, an Integer, and a Dog in the exact same list. When you retrieved the item, you had to blindly cast it back to its original type. If you guessed wrong, your program crashed with a ClassCastException.

Generics (<T>) allow you to parameterize types. You can tell the compiler: "This List is STRICTLY for Dogs." If anyone tries to put a String in it, the code won't even compile.

Type Erasure: The Grand Illusion

Here is the biggest secret about Java Generics: They don't exist at runtime.

For backwards compatibility with older Java versions, the compiler uses Type Erasure. It verifies your types during compilation, but when it creates the Bytecode, it literally erases the <String> part and turns everything back into an Object, automatically inserting invisible casts for you.

Java
// What you write:
List<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0);

// What the JVM actually runs (Type Erasure):
List list = new ArrayList();
list.add("Hello");
String s = (String) list.get(0); // Invisible cast injected by compiler!

Wildcards and PECS

Generics are invariant. A List<Dog> is NOT considered a subclass of List<Animal>. To allow flexibility in method parameters, we use Wildcards (?).

The golden rule for Wildcards is PECS: Producer Extends, Consumer Super.

  • Producer Extends (<? extends Animal>): Use this when you want to READ from the collection. You are guaranteed to get an Animal, but you can't add to it because you don't know the exact specific subclass.
  • Consumer Super (<? super Dog>): Use this when you want to WRITE to the collection. You can safely insert a Dog, but reading from it is dangerous because you don't know what upper-level object it might actually be.
JavaPECS in Action
// PRODUCER EXTENDS (Reading only)
public void feedAnimals(List<? extends Animal> animals) {
    for (Animal a : animals) {
        a.eat(); // Safe to read
    }
    // animals.add(new Dog()); // ERROR! The list might actually be a List<Cat>!
}

// CONSUMER SUPER (Writing only)
public void addDogs(List<? super Dog> list) {
    list.add(new Dog()); // Safe to write (List could be List<Dog>, List<Animal>, List<Object>)
    // Animal a = list.get(0); // ERROR! We don't know if the list holds raw Objects!
}

Real-World Use Case: Generic Repositories

In large enterprise frameworks like Spring Data JPA, you don't write a separate database access class for every single entity. You write one GenericRepository<T>. The framework allows you to say public interface UserRepository extends GenericRepository<User>, and the compiler instantly wires up the Save, Find, and Delete methods to strictly accept and return the User class.

Interview Preparation

Question 1
What is Type Erasure?
It's a process where the Java compiler strips out generic type information after verifying type safety, converting it to raw Object bytecode. This means generics do not exist at runtime.
Question 2
Is List<String> a subclass of List<Object>?
No! Generics are invariant. If they were covariant, you could assign a List<String> to a List<Object> variable, and then accidentally insert an Integer into the String list.
Question 3
Explain the PECS principle.
Producer Extends, Consumer Super. Use ? extends T when you only need to read data (Producer). Use ? super T when you only need to write data (Consumer).
