File I/O and Networking

Build robust file and network operations with NIO, sockets, and modern HTTP client APIs.

TH

Theory Deep Dive

External systems as unreliable boundaries

Mental Model

File systems and networks are slower and less predictable than memory. They can fail because of permissions, missing paths, locks, timeouts, partial writes, unavailable hosts, malformed responses, or resource exhaustion.

Good I/O code is defensive without being noisy. It validates inputs, sets timeouts, closes resources, logs enough context, and translates low-level failures into errors the rest of the application can understand.

Networking adds protocol concerns: request format, response parsing, retry safety, idempotency, authentication, rate limits, and backpressure. A successful connection is only one part of a correct network operation.

Principles That Matter

  • Understand the purpose of the feature before memorizing its syntax.
  • Connect the concept to runtime behavior, API design, and failure cases.
  • Prefer readable code that makes assumptions explicit.
  • Use the feature when it reduces complexity, not just because it exists.

  • Using the feature everywhere instead of where it fits the problem.
  • Skipping the connection between this topic and surrounding Java concepts.

NIO File Operations

Java
Path path = Paths.get("logs/app.log");
Files.createDirectories(path.getParent());
Files.writeString(path, "service-started\n", StandardOpenOption.CREATE, StandardOpenOption.APPEND);

String content = Files.readString(path);
List<String> lines = Files.readAllLines(path);

Prefer java.nio.file.Files over old java.io.File APIs in new code.

    .uri(URI.create("https://api.example.com/health"))
    .GET()
    .build();

HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.statusCode());

Socket Basics

For custom protocols, TCP sockets provide full control over byte-level communication.

Java
try (Socket socket = new Socket("localhost", 8080);
     BufferedWriter out = new BufferedWriter(new OutputStreamWriter(socket.getOutputStream()));
     BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()))) {

    out.write("PING\n");
    out.flush();

    String reply = in.readLine();
    System.out.println(reply);
}

Reliability Practices

  • Always set connect/read timeouts for network calls.
  • Use retry with backoff only for idempotent operations.
  • Close resources with try-with-resources.
  • Use buffering for large I/O throughput improvements.
  • Capture structured logs on failed I/O for root-cause analysis.
